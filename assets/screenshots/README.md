# Screenshots

This folder hosts every image referenced from the project [`README.md`](../../README.md) gallery.

> Binary assets are intentionally **not** generated by the build process — drop the captures here yourself when you have a working environment to record from. The repository is wired up so the gallery renders the moment a file with the expected name appears.

## Required captures

Capture each screen at **1440 × 900** in light mode unless noted otherwise. Save as PNG (8-bit RGBA) optimized through [TinyPNG](https://tinypng.com) or `pngquant` so each file stays under ~250 KB.

| Filename | Page | Notes |
|---|---|---|
| `hero-landing.png` | `/` (landing) | Used as the hero image at the top of the README. |
| `landing.png` | `/` (landing) | Same page, gallery thumbnail. |
| `catalog.png` | `/courses` | Show the filter bar + at least 6 cards. |
| `course-detail.png` | `/courses/:slug` | Hero + curriculum panel visible. |
| `lesson-player.png` | `/learn/:slug/:lessonId` | Video playing, sidebar curriculum visible. |

| `certificate.png` | Certificate preview / download | Show the rendered PDF or modal. |

| `mobile-overview.png` | Any page at **375 × 812** | iPhone-sized portrait crop. |
| `lighthouse.png` | Lighthouse report export | Highlights from the Performance section. |

## Naming rules

- Use only the filenames above so the README links don't have to be edited.
- All lowercase, hyphen-separated, no spaces, no version suffixes.
- Keep the source captures (PSD, Figma frames) **outside** this folder — only the published assets ship to the public repo.
